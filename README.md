# 🏥 מערכת ניהול מרפאה - Clinic Management System

מערכת מלאה (Full-Stack) לניהול מרפאה, המאפשרת ניהול לקוחות, מעקב ביקורים, ניתוח עומסי מחלקות והפקת דוחות מתקדמים. המערכת בנויה בארכיטקטורת שרת-לקוח מופרדת לחלוטין (Decoupled Architecture).

---

## 🏗️ ארכיטקטורת המערכת ומבנה הפרויקט

המערכת מחולקת לשלוש שכבות עיקריות המפרידות בין לוגיקת השרת, חוקי העסק ותצוגת המשתמש:

1. **Server (Web API):** שרת RESTful המבוסס על ASP.NET Core, האחראי על גישה לבסיס הנתונים (SQL Server), הרצת פונקציות ופרוצדורות מורכבות, וחשיפת קצוות (Endpoints) מאובטחים.
2. **Core / Entities & DTOs:** שכבת קוד משותפת (Class Library) המגדירה את המודלים של מסד הנתונים, ישויות ה-Entity Framework ורכיבי ה-DTO (Data Transfer Objects) לצורך העברת נתונים נקייה בין השרת לקליינט.
3. **Client (WPF App):** אפליקציית שולחן עבודה מודרנית מבוססת XAML ו-C# המציגה ממשק משתמש דינמי, עשיר ומהיר מול המשתמשים השונים.

---

## 🛠️ טכנולוגיות מרכזיות

### צד שרת (Backend)
* **Framework:** ASP.NET Core Web API
* **Database ORM:** Entity Framework Core (EF Core)
* **Database:** SQL Server (כולל שימוש ב-Scalar Functions ו-Stored Procedures לייעול שאילתות)
* **Architecture Patterns:** Repository Pattern & Service Layer

### צד לקוח (Frontend / Desktop)
* **UI Framework:** WPF (Windows Presentation Foundation)
* **Markup:** XAML (e.g., DataGrid, DataTemplate, WrapPanel Control)
* **Communication:** `HttpClient` המבצע קריאות אסינכרוניות (`async/await`) וסריאליזציה מובנית עם `GetFromJsonAsync`.

## 🌟 פיצ'רים מרכזיים במערכת

### 🔑 מנגנון ניתוב והתחברות (Login Routing)
* **מנהל מערכת (Admin):** כניסה באמצעות קוד ייעודי (123) הפותחת ממשק ניהול שליטה מלא.
* **רופא (Doctor):** זיהוי מזהה רופא ופתיחת לוח זמנים יומי, פירוט תורים מתוכננים וסטטיסטיקות אישיות.
* **לקוח (Client):** התחברות לקוח המציגה את פרטיו האישיים ואת תאריך הביקור האחרון שלו במרפאה בזמן אמת.

### 👥 ממשק ניהול אדמין (Admin Dashboard)
* **ניהול לקוחות מלא (CRUD):** הצגת כלל הלקוחות ב-DataGrid דינמי, עריכת פרטי לקוח ישירות מהממשק, ומחיקת לקוחות תוך שמירה על שלמות נתונים קשיחה (מניעת מחיקה של לקוח עם היסטוריית ביקורים קיימת).
* **ייבוא וייצוא נתונים (Excel Integration):**
  * ייבוא מסיבי של לקוחות חדשים למערכת מתוך קובץ אקסל (`.xlsx`).
  * הפקה וייצוא של דוח שעות רופאים ממוצע חודשי ישירות לקובץ אקסל מקומי.
* **דפדוף נתונים (Pagination):** מימוש מנגנון שליפת נתונים יעיל בצד הלקוח באמצעות לולאות `while` חכמות השואבות את עמודי המידע מהשרת בהדרגה עד להשלמת הרשימה.
* **שאילתות אנליטיות מבוססות DB:**
  * **עומס מחלקות:** שליפת אחוזי העומס הנוכחיים בכל מחלקה רפואית.
  * **עיר בעדיפות:** חישוב תרחישי עדיפויות גיאוגרפיים מבוססי תאריך ספציפי.

## ⚙️ דרישות קדם והרצה

### דרישות מערכת
* .NET 6.0 SDK / .NET 8.0 SDK ומעלה
* Visual Studio 2022
* SQL Server LocalDB או SQL Server Instance פעיל

### שלבי הרצה ראשונית
1. **הגדרת בסיס הנתונים:**
   * עדכני את מחרוזת החיבור (`ConnectionString`) בקובץ `appsettings.json` בצד השרת.
   * הרצי את ה-Migrations או הריצי את סקריפט ה-SQL הכולל את הפונקציה `dbo.GetLastVisitToClient`.
2. **הרצת השרת:**
   * הגדירי את פרויקט ה-Web API כפרויקט הסטארט-אפ והריצי אותו (מומלץ לוודא שה-Swagger עולה כהלכה).
3. **הרצת האפליקציה (WPF):**
   * ודאי שכתובת ה-Base Address של ה-`HttpClient` בשירותים השונים מתאימה לפורט של השרת המקומי שלך.
   * הרצי את פרויקט ה-WPF והתחברי למערכת.

## 🔒 טיפול בשגיאות ויציבות (Resilience)
* המערכת מטפלת בצורה מובנית בשגיאות המרת ערכי `DBNull.Value` מה-Database לטיפוסי `Nullable<DateTime>` ב-C# על מנת למנוע קריסות (כגון שגיאות 500) עבור לקוחות חדשים ללא היסטוריית ביקורים.
* חלוניות דיאלוג (`MessageBox`) מציגות חיווי מפורט למשתמש במקרה של בעיות תקשורת רשת או הפרת שלמות נתונים (Foreign Key Constraints) בעת ניסיונות מחיקה.
